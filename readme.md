### Пояснение алгоритма:

1. **Моделирование** **: Коридор представляется как граф, где вершины **`0..n`, а рёбра идут только вперёд на `+1` или `+2`. Посещать можно только точки из массива `valid`.
2. **Прямой BFS (`dist_f`)** **: Находит минимальное количество прыжков от старта до каждой платформы. Если **`dist_f[n] == -1`, путь невозможен.
3. **Обратный BFS (`dist_b`)** **: Находит минимальное количество прыжков от каждой платформы до финиша.**
4. **Восстановление пути** **: Идём от **`0` к `n`. На каждом шаге проверяем прыжки `1` и `2` по порядку. Если прыжок `jump` в точку `nxt` удовлетворяет условию `dist_f[cur] + 1 + dist_b[nxt] == min_jumps`, значит он принадлежит **какому-то** кратчайшему пути. Поскольку мы перебираем `1` перед `2`, мы гарантируем лексикографическую минимальность результата.
5. **Сложность** **: **`O(n)` по времени и памяти. При `n ≤ 10`

Робот-курьер спешит доставить важный микрочип на исследовательскую станцию, но путь проходит через аварийный коридор. Пол в коридоре частично разрушен, и наступать можно только на надёжные платформы, расположенные в некоторых точках. Обходной путь слишком долгий, поэтому роботу нужно двигаться только вперёд, прыгая по платформам.

Для простоты представим коридор в виде координатной прямой. Робот стартует в точке **0**, а станция назначения находится в точке **n**.

Робот может прыгать только на **1** или **2** метра вперёд. То есть из точки с координатой **x** он может попасть только в точки **x**+**1** и **x**+**2**. Зная координаты надёжных платформ, робот хочет определить минимальное количество прыжков, необходимое, чтобы добраться из точки **0** в точку **n**.

Кроме того, среди всех путей с минимальным числом прыжков робот хочет выбрать  *лексикографически минимальный* . Путь задаётся последовательностью чисел **1** и **2**, где **i**-е число обозначает длину **i**-го прыжка.

Помогите роботу найти оптимальный маршрут.

## Формат ввода

В первой строке находятся два числа **n**, **k** (**1**≤**n**≤**1**0**5**, **0**≤**k**<**n**) — координата точки назначения и количество надёжных платформ соответственно.

Во второй строке через пробел записаны **k** чисел **x**i (**1**≤**x**i≤**n**) — координаты платформ.

Гарантируется, что координаты платформ заданы в строго возрастающем порядке: **x**1>**0** и для всех **i**>**1** выполнено **x**i>**x**i**−**1****.

## Формат вывода

Если добраться до точки n**n** невозможно, выведите в единственной строке `-1`.

Иначе в первой строке выведите минимальное количество прыжков, необходимое, чтобы попасть из точки **0** в точку **n**.

Во второй строке выведите оптимальный путь — последовательность чисел **1** и **2** без пробелов.

Путь должен иметь минимальную длину, а среди всех таких путей быть лексикографически минимальным.

### Пример

**Ввод**

```
5 3
2 3 4
```

**Вывод**

```
3
212
```
